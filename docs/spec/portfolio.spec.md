# ポートフォリオサイト 仕様書

## コンセプト

「大企業の現場で鍛えた設計力で、スタートアップの課題をスピード解決します」

- ターゲット：個人・スタートアップのクライアント
- トーン：クリーン・ミニマル・信頼感
- 言語：日本語のみ
- 名前：ハンドルネーム or イニシャル表記

---

## 技術スタック

| レイヤー | 技術 |
|---|---|
| フレームワーク | Next.js 14（App Router） |
| スタイル | Tailwind CSS + shadcn/ui |
| アニメーション | Framer Motion（スクロールアニメ） |
| フォーム | React Hook Form + Resend（メール送信） |
| デプロイ | Vercel |

---

## デザイン仕様

### カラーパレット

| 用途 | カラー |
|---|---|
| 背景 | `#FFFFFF`（白） |
| サブ背景（セクション区切り） | `#F8F8F8`（薄グレー） |
| テキスト（メイン） | `#111111` |
| テキスト（サブ） | `#666666` |
| アクセント | `#2563EB`（ブルー）|
| ボーダー | `#E5E5E5` |

### タイポグラフィ

- 日本語：Noto Sans JP
- 英数字：Inter
- 見出し：太め（font-bold）
- 本文：Regular、行間ゆったり（leading-relaxed）

### デザイン方針

- 余白を広めにとってスッキリ見せる
- アニメーションは控えめ（フェードイン程度）
- カードにはうっすらシャドウ（shadow-sm）
- ボタンはシンプルなアウトライン or ソリッド

---

## サイト構成（シングルページ）

全セクションを1ページに縦に並べるSPA構成。
ナビゲーションクリックで該当セクションにスムーズスクロール。

```
/ （シングルページ）
  ├── Header（ナビゲーション固定）
  ├── #hero（ファーストビュー）
  ├── #about（自己紹介）
  ├── #skills（スキル）
  ├── #works（制作物）
  ├── #contact（お問い合わせ）
  └── Footer
```

---

## 各セクション詳細

### Header（固定ナビ）

- 左：ハンドルネーム or イニシャルロゴ（テキストロゴでOK）
- 右：About / Skills / Works / Contact のアンカーリンク
- スクロール時に背景に薄いblur + border-bottom

---

### #hero（ファーストビュー）

**レイアウト：** 中央揃え、画面高さ100vh

**コンテンツ：**
```
（小さく）フルスタックエンジニア
（大きく）業務の課題を、
         コードで解決します。
（サブ）通信大手で4年間、ミッションクリティカルなシステム開発に従事。
       業務自動化・効率化を得意とするフルスタックエンジニアです。
（ボタン）制作物を見る → #worksへスクロール
```

---

### #about（自己紹介）

**レイアウト：** 左にプロフィールカード、右にテキスト（2カラム）

**コンテンツ：**

左カード：
- アバター画像（イラストアイコン or イニシャルアイコン）
- ハンドルネーム
- 肩書き：フルスタックエンジニア
- GitHubリンク

右テキスト：
```
某通信会社でフルスタックエンジニアとして4年間勤務。
ワークフローシステムの設計・開発・改善を担当し、
リリース作業の自動化により作業時間を90%削減した実績があります。

認証システムの移行・WebAPI設計・複数システムとのリアルタイム連携など、
ミッションクリティカルな環境での開発経験が強みです。

現在は副業として、個人・スタートアップ向けの
Webアプリ開発・業務効率化ツールの案件を受け付けています。
```

**ハイライトカード（3枚）：**
- 🏢 本業4年 / 大企業での実務経験
- ⚡ 作業時間90%削減 / 業務自動化の実績
- 🔐 認証・セキュリティ / SSO・WebAPI設計経験

---

### #skills（スキル）

**レイアウト：** カテゴリ別にグループ化したバッジ一覧

**コンテンツ：**

```
フロントエンド
  Angular / Next.js / React / TypeScript / Tailwind CSS

バックエンド
  Django / Python / uWSGI / REST API設計

インフラ・その他
  Nginx / Linux / Git / GitHub / Vercel / Supabase

開発スタイル
  AI駆動開発（Claude Code / GitHub Copilot）
  ワークフロー設計 / バッチ処理 / システム間連携
```

**補足テキスト：**
```
GitHub CopilotやClaude Codeを活用したAI駆動開発で、
スピーディーな開発・納品を実現します。
```

---

### #works（制作物）

**レイアウト：** カード3列グリッド（モバイルは1列）

**各カードの構成：**
- スクリーンショット or OGP画像
- アプリ名
- 一言説明（1〜2行）
- 使用技術バッジ
- ボタン：「デモを見る」「GitHubを見る」

**掲載予定アプリ：**

```
1. SaaS管理アプリ
   契約中のSaaSを一覧管理し、月額費用を可視化するツール
   [Next.js] [Supabase] [Recharts] [Tailwind CSS]

2. 情報レコメンドアプリ（制作予定）
   興味・学習進捗に応じて必要な情報を自動収集・要約・提示するAIツール
   [Next.js] [Claude API] [Supabase]

3. 資産管理アプリ（制作予定）
   銀行・証券のCSVを取り込み、資産・収支を可視化するツール
   [Next.js] [Supabase] [Recharts]
```

---

### #contact（お問い合わせ）

**レイアウト：** 中央揃え、シンプルなフォーム

**コンテンツ：**

```
（見出し）お仕事のご相談
（サブ）副業として承っています。お気軽にご連絡ください。
       ※ 返信は2〜3営業日以内を目安にしております。

フォーム項目：
  - お名前
  - メールアドレス
  - ご相談内容（textarea）
  - 送信ボタン
```

送信後：サンクスメッセージをインライン表示（ページ遷移なし）
メール送信：Resend を使用して自分のメールアドレスに転送

## Claude Codeへの指示文

以下をそのままClaude Codeに貼り付けて開発を開始する。

```
Next.js 14（App Router）+ Tailwind CSS + shadcn/ui + Framer Motion を使って、
フルスタックエンジニアのポートフォリオサイトを作成してください。

【デザイン方針】
- カラー：白ベース・薄グレーのセクション区切り・アクセントカラーは #2563EB（ブルー）
- フォント：日本語は Noto Sans JP、英数字は Inter
- 余白は広めにとり、クリーン・ミニマルな印象に
- アニメーション：スクロール時のフェードイン（Framer Motion）を控えめに使用

【サイト構成】
シングルページ（SPA）構成。以下のセクションを縦に並べる。
- Header（固定ナビ：About / Skills / Works / Contact へのスムーズスクロール）
- Hero（キャッチコピー・サブテキスト・CTAボタン）
- About（自己紹介テキスト＋ハイライトカード3枚）
- Skills（カテゴリ別スキルバッジ一覧）
- Works（制作物カードグリッド：デモURL・GitHub・使用技術バッジ）
- Contact（シンプルなお問い合わせフォーム）
- Footer

【各セクションの内容】

■ Hero
- 小見出し：フルスタックエンジニア
- 大見出し：業務の課題を、コードで解決します。
- サブテキスト：通信大手で4年間、ミッションクリティカルなシステム開発に従事。業務自動化・効率化を得意とするフルスタックエンジニアです。
- CTAボタン：「制作物を見る」→ #worksへスクロール

■ About
- 左カラム：イニシャルアイコン（丸型）＋ハンドルネーム＋肩書き＋GitHubリンク
- 右カラム：自己紹介テキスト（仕様書参照）
- ハイライトカード3枚：本業4年 / 作業時間90%削減 / 認証・セキュリティ経験

■ Skills
フロントエンド・バックエンド・インフラ・開発スタイルの4カテゴリでバッジ表示

■ Works
カード3列グリッド。各カードに画像・アプリ名・説明・技術バッジ・デモリンク・GitHubリンク。
初期データはダミー（後で差し替えられるようにlib/works.tsでデータ管理）

■ Contact
名前・メール・相談内容のフォーム。送信後はサンクスメッセージをインライン表示。
メール送信処理はapi/contact/route.tsにまとめる（Resend使用を想定した構成に）

【技術要件】
- components/sections/ 配下に各セクションをコンポーネント分割
- 制作物データは lib/works.ts で一元管理
- レスポンシブ対応（モバイル・タブレット・PC）
- Vercelにデプロイできる構成
- ハンドルネームや個人情報は "[ハンドルネーム]" のようなプレースホルダーで記載
```

## 開発の進め方

1. **Claude Codeに上記の指示文を貼り付けて開発開始**
2. **ハンドルネーム・GitHubリンクなどを差し替え**
3. **SaaS管理アプリが完成したらWorksセクションのダミーを差し替え**
4. **Resend登録 → Contact フォームのメール送信を有効化**
5. **Vercelにデプロイして公開URLを取得**
